# 대시보드 — 운영 알림 & 추천 액션 정책

## 개요

대시보드 하단의 "운영 알림 / 추천 액션" 영역은 운영자에게 **즉시 조치가 필요한 경고**와 **데이터 기반 행동 제안**을 제공합니다.

두 영역은 목적이 다르므로 2컬럼으로 분리하여 표시합니다.

| 영역 | 목적 | 성격 |
|------|------|------|
| 운영 알림 | 문제 감지 → 즉시 대응 유도 | 수동적 (시스템이 감지) |
| 추천 액션 | 기회 발견 → 성과 개선 유도 | 능동적 (AI 분석 기반) |

---

## 1. 운영 알림 정책

### 1.1 알림 타입 및 우선순위

| 타입 | 용도 | 색상 | 아이콘 | 우선순위 |
|------|------|------|--------|----------|
| `destructive` | 서비스 장애, 성과 급락 등 즉시 조치 필요 | 빨강 (`red-50/200/600`) | `AlertCircle` | 최상 |
| `warning` | 리소스 부족, 설정 미비 등 주의 필요 | 노랑 (`amber-50/200/600`) | `AlertTriangle` | 중간 |
| `info` | 개선 기회, 일반 안내 | 파랑 (`blue-50/200/600`) | `Info` | 낮음 |

### 1.2 알림 발생 규칙

알림은 아래 **카테고리별 조건**에 해당하면 자동으로 생성됩니다.

---

#### 카테고리 A: 서비스 연동 상태

대시보드 "서비스 상태" 영역에 등록된 모든 서비스에 대해 적용됩니다.

**대상 서비스**: 카페24, 인스타그램, 유튜브, 채팅상담, 헤이데어, 마케팅 수신 동의

| 조건 | 타입 | 메시지 패턴 | 액션 |
|------|------|-----------|------|
| 연동 토큰 만료 | `destructive` | "{서비스명} 연동이 만료되었습니다. 재연결이 필요합니다." | 서비스 연결 → `/settings/integrations` |
| API 응답 오류 (연속 3회 이상) | `destructive` | "{서비스명} 서비스에 오류가 발생했습니다. 상태를 확인해주세요." | 서비스 연결 → `/settings/integrations` |
| 연동 토큰 만료 임박 (D-7 이내) | `warning` | "{서비스명} 연동이 {N}일 후 만료됩니다. 갱신이 필요합니다." | 서비스 연결 → `/settings/integrations` |
| 서비스 미연동 상태 | `info` | "{서비스명}을 연동하면 마케팅 채널을 확장할 수 있습니다." | 서비스 연결 → `/settings/integrations` |

> `{서비스명}`은 실제 해당 서비스명으로 치환됩니다. 여러 서비스에 동시에 문제가 발생하면 각각 개별 알림으로 표시됩니다.

#### 카테고리 B: 포인트 / 리소스

### 1.5 메시지 템플릿 규칙

모든 알림 메시지는 아래 패턴을 따릅니다.

| 요소 | 규칙 | 예시 |
|------|------|------|
| 치환 변수 | `{변수명}`으로 표기, 런타임에 실제 값으로 치환 | `{서비스명}`, `{캠페인명}`, `{N}` |
| 문장 구조 | **현상 서술 + 행동 유도** (2문장 이내) | "~ 만료되었습니다. 재연결이 필요합니다." |
| 수치 포함 | 임계값 관련 알림은 반드시 현재 수치를 포함 | "잔여 포인트가 **820P**입니다." |
| 톤 | `destructive`: 긴급 / `warning`: 권유 / `info`: 안내 | "~ 중단될 수 있습니다" / "~ 필요합니다" / "~ 높여보세요" |

### 1.6 표시 규칙

- 최대 표시 개수: **5개** (초과 시 우선순위 순으로 상위 5개만 표시)
- 정렬 순서: `destructive` → `warning` → `info` (동일 타입 내에서는 최신순)
- 동일 카테고리 중복 방지: 같은 대상에 대해 여러 임계값에 해당하면 **상위 타입만** 표시
- 알림이 0개일 때: "현재 알림이 없습니다." 메시지 표시
- 각 알림에는 반드시 **액션 버튼**(이동 경로)을 포함

## 2. 추천 액션 정책

### 2.1 추천 카드 구조

각 추천 카드는 아래 4가지 요소를 필수로 포함합니다.

| 요소 | 설명 | 작성 규칙 |
|------|------|----------|
| `title` | 현재 상황 진단 | 부정형 진술 ("~이 낮습니다", "~이 없습니다", "~을 놓치고 있습니다") |
| `description` | 데이터 근거 + 해결 방법 | 반드시 수치 근거 포함 (기간, 대상 수, 비교 기준) |
| `impact` | 예상 개선 효과 | "예상 ~" 접두사 + 정량 수치 (%p, 건수, 매출) |
| `action` | 즉시 실행 가능한 CTA | 동사형 행동 ("문구 수정", "캠페인 만들기", "설정") |

### 2.2 추천 발생 규칙

추천은 아래 **카테고리별 조건**을 평가하여 자동 생성됩니다.

---

#### 카테고리 A: 성과 지표 저조

현재 성과가 기준치(업종 평균 또는 자체 이전 기록)보다 낮을 때 발생합니다.

| 조건 | 제목 패턴 | 설명 패턴 | 효과 패턴 |
|------|----------|----------|----------|
| 클릭률 < 업종 평균 | "{지표명}이 평균보다 낮습니다" | "현재 {지표} {현재값}은 업종 평균 {기준값}보다 낮습니다. {개선방법}을 통해 개선할 수 있습니다." | "예상 {지표} +{수치} 개선" |
| 전환율 < 업종 평균 | "{지표명}이 평균보다 낮습니다" | (동일 패턴) | (동일 패턴) |
| 열람률 < 업종 평균 | "{지표명}이 평균보다 낮습니다" | (동일 패턴) | (동일 패턴) |

> 비교 기준: 업종 평균 → 자체 최근 30일 평균 → 전주 동기간 순으로 우선 적용

---

#### 카테고리 B: 미활용 기회

설정 가능하지만 아직 사용하지 않는 기능/캠페인이 있을 때 발생합니다.

| 조건 | 제목 패턴 | 설명 패턴 | 효과 패턴 |
|------|----------|----------|----------|
| 특정 고객군 대상 캠페인 없음 | "{고객군} 전용 캠페인이 없습니다" | "지난 {기간}간 {고객군} {N}명 중 캠페인 노출은 0건입니다. {제안 캠페인}을 추가하면 {기대효과}." | "예상 {지표} +{수치} 증가" |
| 이탈 고객 리마인드 미설정 | "{이탈유형} 고객을 놓치고 있습니다" | "최근 {기간}간 {이탈유형} 고객 {N}명에게 리마인드 메시지를 보내지 않았습니다. {캠페인유형} 캠페인은 평균 {전환율} 구매 전환을 기록합니다." | "예상 추가 매출 약 {N}건" |

> **고객군 예시**: 신규 방문자, 재방문 고객, VIP 고객, 휴면 고객
> **이탈유형 예시**: 장바구니 이탈, 결제 페이지 이탈, 회원가입 미완료

#### 카테고리 C: 트렌드 감지

성과 지표의 추세 변화를 감지했을 때 발생합니다.

| 조건 | 제목 패턴 | 설명 패턴 | 효과 패턴 |
|------|----------|----------|----------|
| 지표 N일 연속 하락 (N ≥ 3) | "{지표명}이 {N}일 연속 하락 중입니다" | "{기간} 동안 {지표}가 {시작값}에서 {현재값}으로 하락했습니다. {개선방법}을 권장합니다." | "예상 {지표} 회복" |
| 특정 시간대/요일 전환 집중 | "{시간/요일}에 {지표}이 {N}배 높습니다" | "{분석기간} 기준 {시간/요일} {지표}가 평균 대비 {N}배 높습니다. 해당 시간대 집중 발송을 추천합니다." | "예상 {지표} +{N}% 증가" |
| 재방문/재구매 고객 증가 추세 | "{고객군}이 증가하고 있습니다" | "최근 {기간}간 {고객군}이 {증가율} 증가했습니다. {제안 캠페인}으로 {기대효과}." | "예상 {지표} 증가" |

### 2.3 표시 규칙

- 최대 표시 개수: **3개** (예상 효과가 큰 순서로 정렬)
- 라벨: "AI 분석 기반" 표시 (Lightbulb 아이콘 + indigo 컬러)
- 각 카드에는 반드시 **정량적 예상 효과**(impact)를 포함
- 예상 효과는 초록색 텍스트 + TrendingUp 아이콘으로 강조
- CTA 버튼: indigo 배경의 실행 버튼
- 동일 카테고리에서 최대 2개까지만 표시 (다양성 확보)

### 2.4 문구 작성 원칙

1. **제목**: 현재 상태를 명확하게 진단 (부정형 → "~이 낮습니다", "~이 없습니다")
2. **설명**: 반드시 수치 근거를 포함 (기간, 대상 수, 비교 기준)
3. **효과**: "예상 ~" 접두사 + 구체적 수치 (%p, 건수, 매출)
4. **CTA**: 동사형으로 즉시 실행 가능한 행동 ("문구 수정", "캠페인 만들기", "설정")

### 2.5 치환 변수 목록

| 변수 | 설명 | 예시 |
|------|------|------|
| `{지표명}` | 성과 지표 이름 | 클릭률, 전환율, 열람률 |
| `{현재값}` | 현재 측정 값 | 8.8%, 0.28% |
| `{기준값}` | 비교 기준 값 (업종 평균 등) | 12%, 3.5% |
| `{고객군}` | 대상 고객 세그먼트 | 신규 방문자, 재방문 고객, VIP 고객 |
| `{이탈유형}` | 이탈 단계 | 장바구니 이탈, 결제 페이지 이탈 |
| `{기간}` | 분석 기간 | 7일, 30일, 이번 주 |
| `{N}` | 수치 (건수, 명수, 일수 등) | 187, 2340, 3 |
| `{개선방법}` | 구체적 행동 제안 | 메시지 문구 변경, A/B 테스트, 시간대 변경 |
| `{캠페인유형}` | 제안하는 캠페인 종류 | 웰컴 쿠폰, 장바구니 리마인드, 재방문 혜택 |
