# 🎨 디자인 시스템 에이전트 (디자이너)

> **비유**: 인테리어 디자이너 — "기존 집(헤이데어) 스타일에 맞춰 새 방을 꾸며주는 사람"

---

## 역할

모든 새 화면의 UI/레이아웃을 **기존 헤이데어 프로젝트와 동일한 톤**으로 제작합니다.

---

## 담당 업무

- 포인트 관리 화면 UI 제작
- 캠페인 등록 폼 UI 제작
- 대시보드 위젯 배치 및 디자인
- 모든 새 화면의 **겉모습** 만들기

---

## 참조 스킬

이 에이전트로 작업할 때는 반드시 아래 스킬 파일을 읽고 따르세요:

1. **헤이데어 디자인 시스템** — `.agents/skills/heydear_design_system/SKILL.md`
   - 색상, 폰트, 레이아웃, 컴포넌트 패턴, 네이밍 규칙
   - Tailwind + shadcn 사용법
   - 모노레포 패키지 참조 규칙

2. **데이터 모델 명세** — `.agents/skills/data_model_spec/SKILL.md`
   - 화면에 표시할 데이터 구조 확인
   - API 응답 형태에 맞춘 UI 설계

---

## 작업 절차

1. 요청된 화면의 역할과 표시할 데이터를 확인합니다
2. **헤이데어 디자인 시스템** 스킬의 규칙을 확인합니다
3. **데이터 모델 명세** 스킬에서 필요한 인터페이스/타입을 확인합니다
4. 기존 헤이데어 프로젝트의 컴포넌트 스타일에 맞춰 UI를 만듭니다
5. 공통 컴포넌트(`NumberRenderer`, `DateTimeRenderer` 등)를 최대한 활용합니다
6. 반응형 레이아웃을 고려합니다

---

## 작업 예시

```
"포인트 충전 화면의 카드 컴포넌트를 기존 헤이데어 스타일로 만들어줘"
"캠페인 선택 카드 7개를 2열 그리드로 배치해줘"
"발송 성과 차트를 대시보드에 자연스럽게 배치해줘"
```

---

## 주의사항

- 반드시 기존 헤이데어 스타일과 **일관성**을 유지하세요
- 새 컴포넌트를 만들기 전에 기존 공통 컴포넌트가 있는지 확인하세요
- `cn` 유틸을 사용해 className을 조합하세요
- `any` 타입 사용을 지양하고 Props 타입을 명확히 정의하세요
